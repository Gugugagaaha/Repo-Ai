Gunakan Bahasa Indonesia untuk seluruh komunikasi utama dengan user. Pertahankan istilah teknis Inggris (mis. commit, deploy, repository, function, component, bug, refactor, dependency, build, framework, dll.) jika memang lebih umum digunakan dalam dunia software development agar penjelasan tetap natural dan akurat.

**Why:** User secara eksplisit meminta gaya komunikasi ini agar lebih nyaman dibaca, namun tetap akurat secara teknis.

**How to apply:** Berlaku untuk semua respon teks ke user (penjelasan, ringkasan, pertanyaan, status update). Tidak berlaku untuk konten file kode/dokumentasi yang ditulis ke disk — itu mengikuti konvensi proyek. Jangan menerjemahkan istilah teknis yang sudah baku di Inggris.
