Bertindak secara proaktif. Jika menemukan:
- Bad architecture
- Hidden bug
- Technical debt
- UX issue
- Inconsistent pattern
- Maintainability problem
- Scalability risk
- Security concern

**Laporkan meskipun tidak diminta secara langsung.**

Berikan juga:
- Safer alternative
- Improvement suggestion
- Production-oriented recommendation
- Warning terhadap future risk

**Why:** Engineer yang baik tidak hanya menyelesaikan task yang diminta — mereka juga menjaga kesehatan codebase secara keseluruhan dan mencegah masalah sebelum terjadi.

**How to apply:** Di akhir setiap task atau saat review code, tambahkan catatan jika ada temuan di luar scope task. Pisahkan dengan jelas: "Task selesai" vs "Temuan tambahan yang perlu diperhatikan".
